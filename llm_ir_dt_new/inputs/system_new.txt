The environment is a Dockerized incident response digital twin that emulates a small segmented IT infrastructure with two networks: a client network (10.0.1.0/24) and a server network (10.0.2.0/24). These two networks are connected through a gateway container that functions both as a router and as a Snort IDS sensor.

The gateway container has IP address 10.0.1.10 on the client network and 10.0.2.10 on the server network. It runs Snort IDS for traffic monitoring and iptables for network control. A client container with IP address 10.0.1.11 resides on the client network and has network utilities installed, including nmap, hydra, curl, smbclient, and sshpass. These utilities can perform network scanning, password guessing, HTTP requests, SMB access, and SSH login attempts. Traffic generated by these utilities should be interpreted based on the alerts and observed behavior, not assumed to be benign.

The server network contains five servers. server_ssh at 10.0.2.11 provides OpenSSH. server_samba at 10.0.2.12 provides Samba file-sharing services. server_shellshock at 10.0.2.13 runs Apache with CGI support. In addition, server_web1 at 10.0.2.14 and server_web2 at 10.0.2.15 run Nginx and OpenSSH as additional web servers.

The gateway monitors traffic flowing between the client and server networks.
